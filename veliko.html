But du Projet

Ce projet a pour objectif de créer une carte interactive affichant toutes les stations Vélib' d'Île-de-France. Les utilisateurs peuvent visualiser le nombre de vélos disponibles, qu'ils soient classiques ou électriques, dans chaque station. Une fonctionnalité permet également aux utilisateurs d'ajouter leurs stations préférées à une liste de favoris pour un accès rapide. Enfin, le projet inclut la gestion de deux espaces distincts : un espace client permettant à chaque utilisateur de gérer ses favoris et son compte, et un espace administrateur pour gérer l'ensemble des stations et les données.

Informations sur le Projet

  • Langages utilisés : TWIG, PHP, JAVA
  • Framework utilisé : EduFrame, Symfony
  • Base de données : MySQL
  • API intégrées : API Vélib', OpenStreetMap
  • Date du projet : 2024 - 2025

Difficultés Rencontrées

Gestion des API

L'utilisation de l'API Vélib' a présenté plusieurs défis. Il a fallu garantir que l'API soit intégrée correctement dans la carte, mais aussi accessible au sein de l'espace utilisateur. Et synchronisé les données entre les deux.

Base de Données

- La gestion de la base de données a présenté plusieurs problèmes, notamment l'optimisation des requêtes.
- La synchronisation des données entre les différentes parties de l'application. (carte, espace client, espace administrateur)

CRUD

Le développement des espaces clients et administrateurs a posé des problèmes au niveau du CRUD. Il a fallu créer des interfaces utilisateur adaptées à chaque type de rôle tout en garantissant la protection des données et la gestion efficace des fonctionnalités comme les favoris et les données des stations.
